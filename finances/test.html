    Color

    The UI uses a neutral palette to allow a range of bright content to jump off the page. A system of Depths is used to place backgrounds on a scale from closest (1) to farthest (4) from the user. Colors are scaled (approximately) to their usage/prominence.

    Layout

    The site uses a traditional centered fixed-width Body (1024px) between a Nav and Footer. Layouts within the body vary to match content. It's desired that sections of the site (Marketing, Resources, Units, Management) are internally consistent. Currently there is no mobile layout.

    Buttons

    Buttons are used for a range of actions across the site. The importance of the action and other elements on the page help decide if the case calls for a primary, primary-small, or secondary button. Buttons can be used on any depth, but toggles have a Depth 2 variant.

    Cards

    Cards are useful for containing a preview of a larger view. We're currently using them as launching pads for other views, but they could also be self-contained nuggets of information. Cards contain descriptive information (Titles, thumbnails, meta data) of the content, and on hover an optional prompt for the action you'll do with the content.

    Icons

    Icons are used to add meaning/context to action or text. Icons should be accompanied by text (either inline or with tooltip). Exceptions are the very few universal icons, X < > ?. Icons are implemented with SVG for css styling, flexibility, and scalability goodness. Standard icons are 24x24, but larger and smaller dimensions will be added as needed.

    Modals

    Modals are used for prompts that require the user's full attention, or are best not done inline. They have a title, content, an action, and a close button. Forms can have simple commands or full-featured forms.

    Tables

    Tables are for tabular data (some of this document is in tables, I don't know why. It made sense at the time but its a bad thing). We're not using actual the actual table element because we ran into weird layout problems. However we're keeping the helpful sementatics of table rows, header rows, and especially cells. Most tables have an action column at the far right that could be as simple as a navigation arrow or a handful of action icons. There is a Depth 2 table variant.
